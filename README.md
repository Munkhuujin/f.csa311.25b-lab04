** ArrayIntQueue.java дээрх засварууд:

1. isEmpty() функцийн логик алдаа
Өмнө нь: return size >= 0; гэсэн байсан. Энэ нь size үргэлж 0-ээс их эсвэл тэнцүү байх тул дараалал дотор элемент байсан ч үргэлж "хоосон" гэсэн утга буцааж байв.

Зассан нь: return size == 0; болгож өөрчилсөн. Ингэснээр дараалал яг 0 элементтэй үед л "хоосон" (true) гэж зөв мэдээлнэ.

2. dequeue() болон peek() дээрх хоосон тохиолдлын шалгалт
Өмнө нь: Дараалал хоосон байх үед шууд элемент рүү хандах гэж оролддог байсан нь алдаа гарахад хүргэж байв.

Зассан нь: Функцийн эхэнд if (isEmpty()) { return null; } гэсэн нөхцөлийг нэмж өгсөн. Энэ нь хоосон дарааллаас элемент авах гэж оролдох үед программ гацалгүйгээр аюулгүйгээр null утга буцаадаг болгосон.

3. ensureCapacity() буюу массивыг томсгох үеийн индексжүүлэлт
Өмнө нь: Дараалал дүүрэх үед массивын хэмжээг томсгохдоо элементүүдийг хуучин байрлалаар нь шууд хуулж байсан. Гэвч энэ нь "Circular Array" (дугуй массив) бүтэцтэй тул элементүүдийн логик дараалал алдагдахад хүргэдэг байв.

Зассан нь: Шинэ массив руу элементүүдийг хуулахдаа head индексээс эхлэн дарааллын дагуу (head + i) % oldCapacity томьёогоор нэг бүрчлэн зөв байранд нь хуулж, head индексийг буцааж 0 болгож тохируулсан.
4. testDequeueEmptyQueue() тестийг шинээр бичиж нэмсэн. Энэ тест нь дээрх зассан return null; хэсгийг ажиллуулж шалгасан.


** IntQueueTest.java файл дээр:

1. setUp() функцийг уян хатан болгосон
Өөрчлөлт: mQueue = new ArrayIntQueue(); болон mQueue = new LinkedIntQueue(); хоёрын хооронд шилжиж шалгах боломжтой болгож зассан.

Зорилго: Удирдамжийн дагуу нэг тестээр хоёр өөр бүтцийг (Array болон Linked) зэрэг шалгах боломжийг бүрдүүлсэн.

2. Хоосон дарааллын тестийг нэмсэн (testDequeueEmptyQueue)
Өөрчлөлт: Анхны тестүүдэд дараалал хоосон байх үед dequeue() хийвэл юу болохыг шалгасан тохиолдол байгаагүй. Энэ тестийг нэмснээр ArrayIntQueue-ийн dequeue() доторх return null; гэсэн мөрийг ажиллуулж, JaCoCo дээрх улаан мөрийг ногоон болгосон.

@Test
public void testDequeueEmptyQueue() {
    assertTrue(mQueue.isEmpty());
    assertNull(mQueue.dequeue());
}

3. Массивыг томсгох тестийг баяжуулсан (testEnsureCapacity)
Өөрчлөлт: Анхны 10 гэсэн хэмжээнээс илүү олон (жишээ нь 15) элемент нэмж шалгах хэсгийг нэмсэн.

Зорилго: ArrayIntQueue доторх ensureCapacity() функц бодитоор ажиллаж, массивыг амжилттай томсгож байгааг баталгаажуулсан.

4. "Wrap-around" буюу эргэх тохиолдлыг шалгах тест нэмсэн
Өөрчлөлт: testEnsureCapacityWithWrapAround() тестийг нэмсэн.

Зорилго: Дарааллын head индекс массивын төгсгөл хэсэгт очсон үед (Circular structure) массив томорвол өгөгдөл алдагдаж байгаа эсэхийг нарийн шалгасан. Энэ нь ensureCapacity() доторх индекс тооцооллын алдааг илрүүлэхэд тусалсан.

5. testContent функцийг ажиллагаанд оруулсан
Өөрчлөлт: src/test/resources/data.txt файлаас өгөгдөл уншиж, дараалалд хийгээд буцааж гаргаж ирэхэд дараалал алдагдахгүй байгааг шалгасан.

Зорилго: Бодит өгөгдөл дээр дарааллын (Queue) үндсэн зарчим болох FIFO (First-In, First-Out) зөв ажиллаж байгааг баталгаажуулсан.
